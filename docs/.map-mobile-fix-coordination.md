# Mobile map fix — 3-agent coordination

Bug: On mobile, the connect map "pops out" of its container, overlaps the reviews section header, and allows unwanted touch-drag panning.

---

## Agent-A-CSS

### Diagnosis (Agent A)

Read `ConnectMap.astro` and `index.astro` markup. Structure:

- `.connect-section` > `.connect-split` (grid) > `.connect-info` + `.connect-map`
- `.connect-map` > `.connect-map__viewport` (#connect-map, Leaflet target) + `.connect-map__attribution`
- Reviews live in `.page-dark-zone` immediately after `.connect-section`

Root CSS causes of overlap / "pop out":

1. `.connect-map` and `.connect-map__viewport` had `overflow: visible`, so Leaflet tile/marker layers could paint outside the map box during pan/transform.
2. `.connect-section` had no stacking context or overflow clip — Leaflet control layers use high internal z-index (1000) and could visually sit above the following reviews block.
3. Mobile layout used `min-height: clamp(...)` only (no fixed height), so `invalidateSize()` + touch pan could expand or shift perceived map bounds.
4. `.reviews-section` had no explicit `z-index` / `isolation` to win the boundary against connect.

Touch-drag panning is primarily Leaflet config (Agent B); CSS `touch-action: pan-y` on mobile lets vertical page scroll win over horizontal map drag.

### Prior work (partial — likely aceee725)

Already present before this pass:

- `.connect-section`: `position: relative`, `isolation: isolate`
- `.connect-map`: `overflow: hidden`, `isolation: isolate`, `contain: layout paint`
- `.connect-map__viewport`: `overflow: hidden`
- Mobile: `.connect-map` fixed `height` + `min-height: 0`; `.leaflet-container { touch-action: pan-y }`

### Planned fix (Agent A)

- Finish stacking: `z-index: 1` on connect, `z-index: 2` on reviews.
- Clip Leaflet paint at every layer: `.leaflet-container { overflow: hidden }`, cap control z-index.
- Mobile: overflow clip on `.connect-section` / `.connect-split`, `contain: layout paint size` + `max-height` on map.

## Agent-C-Integrator

### Integration review

Read Agent A coordination notes and git diffs on `ConnectMap.astro`, `global.css`, and `index.astro` after Agents A/B landed changes.

**Agent A fixed** layout breakout and paint bleed: `overflow: hidden` + `contain: layout paint` on `.connect-map`, clipped `.leaflet-container`, capped Leaflet control z-index, mobile fixed height (`clamp(280px, 52vw, 340px)`), `touch-action: pan-y`, and overflow clip on `.connect-section` / `.connect-split` at ≤768px. Also added stacking on `.connect-section` (`z-index: 1`) and `.reviews-section` (`z-index: 2`).

**Agent B disabled** map drag/zoom handlers on mobile via Leaflet options + runtime `applyConnectMapInteractionMode`, and reinforced bounded sizing with `ResizeObserver`, `whenReady`, and `invalidateSize({ pan: false })` so the map stays inside the CSS height box after lazy init.

**Gap found:** Agent A set `z-index: 2` on `.reviews-section`, but reviews live inside `.page-dark-zone` (sibling of `.connect-section`). `.page-dark-zone` had no `position`/`z-index`, so the whole connect block (`z-index: 1`) still stacked above the dark zone in the root context — `.reviews-section` z-index could not beat connect overflow. **I added** `position: relative; z-index: 2` on `.page-dark-zone` so reviews/footer reliably paint above the map.

**index.astro:** No markup changes needed; existing structure (`.connect-section` then `.page-dark-zone` with `#reviews`) is correct.

### Root causes (combined)

1. **Layout breakout** — `overflow: visible` on map containers + fluid `min-height` only on mobile let Leaflet tiles/controls paint outside the map box.
2. **Stacking** — Leaflet internal z-index (~1000) and connect section stacking without elevating `.page-dark-zone` let map visuals overlap the reviews heading.
3. **Scroll hijacking** — Leaflet `dragging`/`touchZoom` on mobile captured touch gestures meant for vertical page scroll.

### Mobile behavior at 390px

- Connect info stacks above map (`order: 1` / `order: 2`).
- Map box is **280px tall** (`clamp(280px, 52vw, 340px)` → 52vw ≈ 203px, clamp floor = 280px).
- Map is **view-only**: no pan/drag/pinch-zoom; vertical page scroll passes through (`touch-action: pan-y` + disabled handlers).
- Zoom +/- buttons still work; **tap marker** opens popup (Call / Directions); address link above map remains primary “get directions” affordance.
- Map tiles/controls **clip inside** the box; **reviews heading** in `.page-dark-zone` stacks above connect (z-index 2 vs 1).
- Popups may clip at map edges on very small screens (acceptable per Agent A note).
